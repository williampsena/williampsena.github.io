Configurando uma conexão por proxy HTTP para o NPM

Fala galera!

Esta dica é simples, porém é um problema muito comum ao tentar utilizar NodeJs em uma organização onde existe o proxy para acesso à internet.

Já tentou instalar um módulo NPM e recebeu erro de HTTP.

Isto ocorre pois precisamos informar para o NPM o endereço do proxy e os dados de autenticação.

Entre em sua linha de comando e digite o comando abaixo:

Exemplo de endereço: http://meuusuario@minhasenha@192.168.0.1:8080

npm config set http-proxy http://usuario:senha@enderecoproxy:porta
npm config set https-proxy http://usuario:senha@enderecoproxy:porta
npm set strict-ssl false

Caso você não seja necessário usuário e senha para autenticação utilize esta configuração.

Exemplo de endereço: http://192.168.0.1:8080

npm config set http-proxy http://enderecoproxy:porta
npm config set https-proxy http://enderecoproxy:porta
npm set strict-ssl false

Para testar vamos instalar um módulo comum e útil o gulp.

npm install -g gulp

Se você estiver no linux recomendo que utilize o sudo para instalar como administrador

sudo npm install -g gulp

Para confirmar se ocorreu a instalação vamos verificar a versão do gulp, executando o comando abaixo:

gulp -v

Pronto até a próxima!
